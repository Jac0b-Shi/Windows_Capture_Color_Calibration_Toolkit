# Milestone 3: HDR-capable Direct3D 11 Chart Window

## 目标
在主窗口配置色卡，并打开**独立的 D3D11 色卡窗口**作为未来 WGC 捕获的正式对象。本轮不实现 WGC、D3D11 捕获后端、CPU readback、LUT、FP16 解码或连续捕获。

## 关键约束
- 正式 ChartWindow 必须使用 Direct3D 11 SwapChain 渲染，不得仅使用 XAML SolidColorBrush。
- XAML 渲染器只用于主界面 SDR 预览：`XamlChartPreviewRenderer`。
- 所有 DXGI/COM/D3D11 代码集中在 `WgcColorCalibrator.Rendering.Direct3D11` 独立项目，不泄漏到页面和 Core。
- App 项目仅保留 WinUI 页面、`ChartWindow`、`SwapChainPanel` 宿主和窗口生命周期协调。
- 支持 SDR、HDR scRGB、HDR10 三种输出模式；切换模式必须重建 SwapChain。
- HDR 能力探测必须检测系统 HDR 是否真正启用，不静默降级。
- 所有用户可见字符串进入 `en-US` 和 `zh-CN` `.resw`。
- 提交分 5 个检查点，每点 restore/build/test 通过。

## 输出模式

| 模式 | SwapChain Format | DXGI Color Space | 输入语义 |
| --- | --- | --- | --- |
| SDR | `B8G8R8A8_UNORM` | `RGB_FULL_G22_NONE_P709` | sRGB 8-bit |
| HDR scRGB | `R16G16B16A16_FLOAT` | `RGB_FULL_G10_NONE_P709` | 线性 BT.709 / scRGB，1.0 = 80 nits |
| HDR10 | `R10G10B10A2_UNORM` | `RGB_FULL_G2084_NONE_P2020` | PQ/BT.2020，显式 SetColorSpace1 |

- 切换模式时重新创建 SwapChain。
- HDR scRGB 为默认 HDR 模式。
- HDR10 标记为 `Hdr10`（而非 `Hdr10Experimental`），但 UI 仍显示 “HDR10 (Experimental)” 并记录其未经验证的 warning；本轮完整实现 PQ 编码路径，但不声称与原神内部实现相同。

## HDR 能力探测

- `DxgiOutputProbe` 枚举 DXGI Output，根据窗口与显示器矩形交集确定主要输出。
- 使用 `IDXGIOutput6.GetDesc1` 获取 `ColorSpace`、`MinLuminance`、`MaxLuminance`、`MaxFullFrameLuminance`。
- 处理窗口移动、系统 HDR 状态变化、DXGI Factory 失效。
- 系统 HDR 未启用时：
  - UI 显示“系统 HDR 未启用”。
  - 不静默降级到 SDR。
  - 用户可选择：取消 / 使用 SDR / 进入裁剪实验模式。
  - 裁剪实验模式在 `ChartRenderSession.Warnings` 中记录 `hdr-requested-while-output-sdr` 和 `out-of-range-values-may-be-clipped`。

## Tone Mapping

- 接口 `IToneMapper`：
  ```csharp
  public interface IToneMapper
  {
      string Id { get; }
      Vector4 Map(Vector4 linearColor, ToneMappingParameters parameters);
  }
  ```
- 首轮实现：
  - `DirectScRgb`：输入直接作为线性 scRGB，不映射。
  - `ReferenceWhiteScaled`：sRGB 转线性，乘以 `paperWhiteNits / 80`，输出到 FP16 scRGB。
- 不实现 `GenshinApproximation` 等未经验证的映射。

## 颜色输入

1. **SDR Color**：`#RRGGBB`，sRGB 编码；HDR 模式下按 Paper White 转换。
2. **HDR Color**：Linear RGB / scRGB float，可大于 1.0，仅允许用于 HDR 输出模式。
3. `ChartDefinition` 记录：source encoding、target luminance、paper white、peak brightness、tone mapper、requested output mode。
4. `Rgb8` 不再被自动视为 HDR 数值。

## UI 设计

- Chart 页面承载所有 HDR 控制：
  - Output Mode 下拉框（SDR / HDR scRGB / HDR10 (Experimental)）
  - Paper White 数值输入（默认 200 nits）
  - Peak Brightness 数值输入（默认 1000 nits）
  - Exposure EV 输入（默认 0）
  - Tone Mapping Mode 下拉框
  - HDR 能力状态显示
  - 系统 HDR 未启用提示与操作按钮
- ChartWindow 本身不含按钮；仅显示 SwapChain 渲染区域。
- Debug Overlay 在 ChartWindow 中作为可选覆盖层，开启时标记会话不可测量。

## 测试计划

新增测试：
1. sRGB 转线性 RGB 正确。
2. 80 nits → scRGB 1.0。
3. 200 nits → scRGB 2.5。
4. 1000 nits → scRGB 12.5。
5. Exposure EV 缩放正确。
6. SDR 模式拒绝 HDR-only Patch。
7. HDR 未启用时不得静默降级。
8. `requestedOutputMode` 与 `actualOutputMode` 分别记录。
9. SwapChain format / color space 模式映射正确。
10. HDR/SDR 模式切换必须触发 renderer / SwapChain 重建。
11. Paper white > 0 被拒绝。
12. Peak brightness >= paper white。
13. 非有限 float 输入被拒绝。
14. HDR10 PQ 编码公式输出值正确（至少对标准测试点如 80 nits / 1000 nits）。
15. HDR warning 正确写入 `RenderSession`。
16. 物理像素 ↔ DIP 转换在 1.0 / 1.25 / 1.5 / 1.75 / 2.0 下正确。
17. 中英新增资源键集合一致。

不编写依赖桌面交互的自动 UI 测试；D3D11 窗口渲染采用人工验收。
